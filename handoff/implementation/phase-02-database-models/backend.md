Open Issues / Risks
- **DEC-BE-001**: Migration generated and tested against SQLite because PostgreSQL is not reachable from the agent sandbox (same network restriction as DEC-FE-001). The migration file is dialect-agnostic and should apply cleanly to PostgreSQL, but the `user_role` shared enum type may need a manual check on first PG deploy. Logged in decision-log.md.
- **`RoleDisplayName` reuses `user_role` enum**: Both `User.role` and `RoleDisplayName.role` refer to the same PostgreSQL enum type `user_role`. The `RoleDisplayName` column uses `create_type=False` to avoid duplicate creation. This is correct for PG but should be validated on first PostgreSQL deploy.
- **Architecture doc lists individual model files** (e.g. `article.py`, `batch.py`). The doc does not list `article_supplier.py`, `article_alias.py`, `inventory_count.py` (which includes `InventoryCountLine`), or `enums.py`. These were added to match the data model spec. `InventoryCountLine` was placed in `inventory_count.py` since the architecture doc only lists one file for that module.
- **`instance/wms_dev.db`**: SQLite verification DB created in `backend/instance/`. Should be gitignored.
